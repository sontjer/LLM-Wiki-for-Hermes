# 链接规范

## 链接格式

使用 Obsidian 标准的 `[[wikilink]]` 格式：

```
[[文件名]]              → 链接到同名文档
[[文件名|显示文字]]      → 自定义显示文字
```

## 什么时候建链

对每一篇文档，AI 执行以下操作：

### 1. 内容关联链接（重要）

扫描文档全文，当出现以下情况时，在文档末尾追加 `## 关联文档` 段落：

```
## 关联文档

- [[Cloudflare Tunnel配置教程]] — 同一主题的详细配置
- [[Docker部署mihomo]] — 相关技术的实践案例
- [[nftables规则配置]] — 文中引用的前置知识
```

**建链规则：**
- 同一技术栈的文档必须互链
- 文中出现的技术名词、工具名、协议名，如果在 Wiki 中有对应文档，加链接
- 每篇文档至少关联 2 篇、最多 8 篇相关文档

### 2. 索引关联

每篇新文档加入后，自动更新对应分类的 `📑索引/` 页面，加入该文档的链接条目。

## 禁止行为

- 不要修改源文件目录下的任何内容
- 不要删除 `指令/` 中的文件
- 不要创建孤立文档（每篇文档都必须有至少一个入链）
