# 链接规范

## 链接格式双轨制（重要）

LLM Wiki 使用两种链接格式，适用于不同场景：

### 格式 A：关联文档段 — 无分类前缀

```
## 关联文档

- [[古德里安]]              ← 同分类 → [[人物/古德里安]] 的简写
- [[淞沪会战]]              ← 同分类 → [[战役/淞沪会战]] 的简写
```

**规则：** 关联文档段只使用 `[[文件名]]` 格式（不带分类前缀）。
Obsidian 会自动在全库搜索匹配的文件名，不受目录层级影响。

**来源：** 由 `wiki_init.py` 的 `generate_cross_links()` 生成，链接到同分类的其他文档。

### 格式 B：本页提及段 — 有分类前缀

```
## 本页提及的相关条目

- [[战役/斯大林格勒战役]]    ← 跨分类，保留前缀
- [[人物/毛泽东]]            ← 跨分类，保留前缀
```

**规则：** 本页提及段使用 `[[分类/文件名]]` 格式（带分类前缀）。
用于跨分类引用，避免同名实体混淆。

**来源：** 由 `cross_link_wiki.py` 扫描文档全文后自动生成。

### 为什么要双格式

| 格式 | 适用段落 | Obsidian 解析方式 |
|------|----------|-----------------|
| `[[古德里安]]` | 关联文档 | 全库搜索文件名 → 匹配任意子目录 |
| `[[战役/斯大林格勒战役]]` | 本页提及 | 路径匹配 → 分类/文件名 精确匹配 |

`[[人物/东条英机]]` 格式不应用于关联文档段——Obsidian 将其解释为相对路径 `人物/东条英机.md`，如果文件实际在 `wiki/人物/东条英机.md` 且 vault 根是库根，则无法解析。

## 关联文档规则

### 什么情况下建链

1. **同分类关联** — 自动添加（由 wiki_init.py 的 generate_cross_links 执行）
   - 同一分类下所有其他文档自动关联
   - 超过 10 篇时只展示前 10 篇

2. **跨分类关联** — 自动扫描添加（由 cross_link_wiki.py 执行）
   - 扫描全文，识别提及的其他分类实体名称
   - 追加到 `## 本页提及的相关条目` 段落

3. **强制补链** — 人工手动（用于自动扫描无法识别的跨分类关系）
   - 写入 raw/ 源文件（使用 curl WebDAV PUT 而非 FUSE mount）
   - 格式用 `[[人物名]]` 无前缀

### 人物是链接枢纽

历史/社科类知识库中，人物是最核心的链接枢纽：

```
人物 ↔ 战役      ← 必须双向贯通
人物 ↔ 装备      ← 必须双向贯通
人物 ↔ 部队编制   ← 尽量贯通
人物 ↔ 概念      ← 尽量贯通
```

## 交叉链接验证

每次增删改后运行 `check_links.py`：

```bash
python3 check_links.py
```

检测项目：
- 战役→人物链接是否存在
- 人物→战役链接是否存在
- 输出缺失报告供人工补链

## 禁止行为

- 不要修改源文件目录下的任何内容
- 不要创建孤立文档（每篇文档都必须有至少一个入链）
- 不要在关联文档段使用带分类前缀的链接格式
